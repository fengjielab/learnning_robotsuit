## 📁 文件夹用途

这个文件夹专门用于**带实时渲染的训练**，所有生成的文件都会放在这里，不会污染其他目录。

现在脚本已经改成每次运行自动创建一个独立 run：
- 训练开始先写到 `training_runs/active/<run_name>/`
- 正常结束后移动到 `training_runs/completed/<run_name>/`
- 手动中断后移动到 `training_runs/interrupted/<run_name>/`
- 失败时移动到 `training_runs/failed/<run_name>/`

这样新训练不会再和旧训练共用同一个 TensorBoard 日志目录，而且能一眼看出每个 run 的状态。

## 📈 TensorBoard 监控

不要把 `<timestamp>` 原样输入进去。
正确做法是先查看真实 run 名，再把真实目录复制给 TensorBoard。

先列出当前 run：

```bash
cd /home/mfj/robosuite/my_experiments/experiment_versions/2026-03-23_render_training
find ./training_runs/active ./training_runs/completed ./training_runs/interrupted ./training_runs/failed -maxdepth 1 -mindepth 1 -type d | sort
```

然后使用真实目录启动 TensorBoard，例如：

## 🎯 观察重点

训练时观察以下几点：

### 1. 夹爪行为
| 夹爪值 | 状态 |
|--------|------|
| > 0.03 | 张开 |
| < 0.02 | 闭合 |

**正常行为**：
- 接近方块时张开
- 接触方块时闭合
- 举起时保持闭合

**异常行为**（需要调整）：
- 一直张开 → 没学会抓取
- 一直闭合 → 可能陷入局部最优

### 2. 方块高度
- 初始高度：约 0.83 米（桌面）
- 成功高度：> 0.87 米（相对高度 > 0.04 米）

### 3. 成功率趋势
- 0-10 万步：可能为 0%（正常）
- 10-30 万步：开始有成功率
- 30-50 万步：成功率应稳定上升

## 🔧 如果发现问题

### 问题 1：夹爪一直闭合
**可能原因**：模型学会了"作弊"——闭合夹爪获得稳定奖励

**解决方案**：
1. 降低抓取奖励
2. 增加探索率（ent_coef）
3. 重新设计奖励函数

### 问题 2：方块高度不增加
**可能原因**：模型没学会举起动作

**解决方案**：
1. 增加举起奖励权重
2. 增加训练步数
3. 使用课程学习

### 问题 3：成功率波动大
**可能原因**：学习率太高或 batch size 太小

**解决方案**：
1. 降低学习率
2. 增大 batch size
3. 增加 n_epochs
