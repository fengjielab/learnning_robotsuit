# lift.py 通俗解释 - 给小白的完整指南

## 📌 一句话总结

**`lift.py` 定义了一个"机器人举方块"的游戏**——机器人要抓起桌上的红色方块并举到空中。

就像教小孩玩积木："宝宝，把这块红色积木拿起来，举高高！"

---

## 🎮 这个游戏怎么玩？

### 场景长什么样？

```
        🤖 机器人
         │
         │ 手臂
         │
         └─ 🤏 夹爪
              │
              │   红色方块
              │   ┌───┐
              ▼   │███│
    ────────────────────────── 桌子
```

### 游戏规则

1. **开始**：红色方块随机放在桌上某个位置
2. **目标**：机器人用夹爪抓起方块，举到桌面以上 4 厘米
3. **成功**：当方块高度超过桌子 +0.04 米，游戏胜利！
4. **奖励**：成功得 2.25 分

## 📖 完整流程示例

### 运行一个 Lift 环境

```python
import robosuite as suite
import numpy as np

# 1. 创建环境
env = suite.make(
    "Lift",                    # 游戏名称：举方块
    robots="Panda",            # 机器人：Panda
    has_renderer=True,         # 显示窗口
    reward_shaping=True,       # 用过程奖励（更好学）
    control_freq=20,           # 每秒 20 步
    horizon=1000,              # 最多 1000 步
)

# 2. 开始游戏
env.reset()  # 重置，方块随机放桌上

for i in range(1000):
    # 3. 决定动作（这里是随机动作）
    action = np.random.randn(env.action_dim)
    
    # 4. 执行动作
    obs, reward, done, info = env.step(action)
    
    # 5. 显示画面
    env.render()
    
    # 6. 打印信息
    if reward > 0:
        print(f"步{i}: 得到奖励 {reward:.3f}")
    
    if done:
        print("游戏结束！")
        break

env.close()
```

### 奖励是怎么计算的？

```python
# 假设机器人离方块 0.5 米远
dist = 0.5
reaching_reward = 1 - np.tanh(10.0 * 0.5)  # ≈ 0.01 分（很远）

# 机器人靠近到 0.1 米
dist = 0.1
reaching_reward = 1 - np.tanh(10.0 * 0.1)  # ≈ 0.46 分（接近）

# 机器人靠近到 0.02 米
dist = 0.02
reaching_reward = 1 - np.tanh(10.0 * 0.02)  # ≈ 0.82 分（很近）

# 抓住了！
grasping_reward = 0.25  # 固定 0.25 分

# 举起来了！
lifting_reward = 1.0  # 固定 1 分

# 总奖励 = 0.82 + 0.25 + 1.0 = 2.07 分（接近满分 2.25）
```

```
                          ↓

│  动作 (Action)                                           │
│  • 控制机器人移动                                         │
│  • 如：[0.01, 0, 0.02, 0, 0, 0] 表示向前 1cm、向上 2cm    │
└─────────────────────────────────────────────────────────┘
                          ↓

│  奖励 (Reward)                                           │

│  • 奖励设计是强化学习的核心！                              │

                          ↓

│  策略 (Policy)                                           │
│  • 机器人根据状态决定动作的"大脑"                         │
│  • 通过试错学习：什么动作能得到高奖励                      │
└─────────────────────────────────────────────────────────┘
```

### 2. 奖励塑形（Reward Shaping）的重要性

**问题**：如果只在成功时给奖励，机器人可能永远学不会。

```
稀疏奖励的问题：
- 机器人随机乱动 1000 步
- 偶然举起了方块 → 得 2.25 分
- 但概率太低，可能 100 万年都学不会

稠密奖励的好处：
- 靠近方块 → 有奖励
- 抓住方块 → 有奖励
- 举起方块 → 有奖励
- 机器人每一步都知道该往哪走
```

### 3. 模块化编程思想

```
Lift 类不是从零开始的，它继承了：
- SingleArmEnv（单臂环境）
  - ManipulationEnv（操作环境）
    - RobotEnv（机器人环境）
      - 最底层基类

好处：
- 通用功能在父类中实现
- Lift 只关注"举方块"的特定逻辑
- 代码复用，易于维护
```

### 4. 仿真环境的基本组成

```
一个完整的仿真环境 = 
  场景（桌子、地板）
+ 机器人（手臂、夹爪）
+ 物体（方块）
+ 观测（相机、传感器）
+ 奖励函数
+ 成功条件
```

### 奖励函数（核心逻辑）

```python
def reward(self, action=None):
    reward = 0.0
    
    # 检查是否成功
    if self._check_success():
        reward = 2.25  # 成功！得满分
    
    # 如果用奖励塑形
    elif self.reward_shaping:
        # 1. 接近奖励（越近分越高）
        dist = 夹爪到方块的距离
        reaching_reward = 1 - np.tanh(10.0 * dist)
        reward += reaching_reward
        
        # 2. 抓握奖励
        if 抓住了方块:
            reward += 0.25
    
    return reward
```

### 成功检查

```python
def _check_success(self):
    cube_height = 方块的高度
    table_height = 桌子的高度
    return cube_height > table_height + 0.04  # 高出 4 厘米算成功
```
