# demo_control.py 通俗解释

## 📌 一句话总结

这个脚本是一个**控制器测试工具**，它的作用是：**逐个测试每个控制器的每个动作维度，让你直观地看到不同控制器的行为差异**。

---

## 🎯 这个脚本是干啥的？

想象你刚买了一辆新车，你想测试：
- 方向盘向左打多少车会偏多少
- 油门踩多少车会加速多少
- 刹车踩多少车会减速多少

`demo_control.py` 做的就是类似的事情——它测试机器人的"方向盘"和"油门"：

```
测试流程示例（以 OSC_POSE 控制器为例）：

步骤 1: 向 X 方向移动 0.1 米 → 保持 75 步 → 休息 75 步
步骤 2: 向 Y 方向移动 0.1 米 → 保持 75 步 → 休息 75 步
步骤 3: 向 Z 方向移动 0.1 米 → 保持 75 步 → 休息 75 步
步骤 4: 绕 X 轴旋转 0.1 弧度 → 保持 75 步 → 休息 75 步
步骤 5: 绕 Y 轴旋转 0.1 弧度 → 保持 75 步 → 休息 75 步
步骤 6: 绕 Z 轴旋转 0.1 弧度 → 保持 75 步 → 休息 75 步
```

这样你就能直观地看到：**当你给某个方向的动作时，机器人会怎么动**。

## 📊 控制器设置详解

```python
controller_settings = {
    "OSC_POSE":       [6, 6, 0.1],    # 6 维动作，测试 6 次，每次 0.1
    "OSC_POSITION":   [3, 3, 0.1],    # 3 维动作，测试 3 次，每次 0.1
    "IK_POSE":        [6, 6, 0.01],   # 6 维动作，测试 6 次，每次 0.01
    "JOINT_POSITION": [7, 7, 0.2],    # 7 维动作，测试 7 次，每次 0.2
    "JOINT_VELOCITY": [7, 7, -0.1],   # 7 维动作，测试 7 次，每次 -0.1
    "JOINT_TORQUE":   [7, 7, 0.25],   # 7 维动作，测试 7 次，每次 0.25
}
```

### 为什么测试值不同？

| 控制器 | 测试值 | 原因 |
|--------|--------|------|
| OSC_POSE | 0.1 | 操作空间控制，0.1 米/弧度是合理移动量 |
| IK_POSE | 0.01 | 逆运动学更敏感，需要更小的值 |
| JOINT_POSITION | 0.2 | 关节位置控制，0.2 弧度约 11 度 |
| JOINT_VELOCITY | -0.1 | 关节速度，负值是因为约定方向 |
| JOINT_TORQUE | 0.25 | 关节扭矩，需要足够大才能克服重力 |

## 💡 简化版代码示例

如果你想自己写一个简化版的控制器测试，可以这样：

```python
import robosuite as suite
import numpy as np

# 1. 创建环境
env = suite.make(
    env_name="Lift",
    robots="Panda",
    controller_configs=suite.load_controller_config(default_controller="OSC_POSE"),
    has_renderer=True,
    control_freq=20,
    horizon=500,
)

env.reset()

# 2. 定义测试动作序列
test_actions = [
    np.array([0.1, 0, 0, 0, 0, 0]),   # X 方向
    np.array([0, 0.1, 0, 0, 0, 0]),   # Y 方向
    np.array([0, 0, 0.1, 0, 0, 0]),   # Z 方向
    np.array([0, 0, 0, 0.1, 0, 0]),   # 绕 X 旋转
    np.array([0, 0, 0, 0, 0.1, 0]),   # 绕 Y 旋转
    np.array([0, 0, 0, 0, 0, 0.1]),   # 绕 Z 旋转
]

# 3. 执行测试
for i, action in enumerate(test_actions):
    print(f"测试 {i+1}: {action}")
    
    # 执行动作 75 步
    for _ in range(75):
        env.step(action)
        env.render()
    
    # 休息 75 步
    for _ in range(75):
        env.step(np.zeros(6))
        env.render()

env.close()
```

## 🎓 不同控制器的预期行为

| 控制器 | 预期行为 |
|--------|----------|
| **OSC_POSE** | 手爪依次沿 X/Y/Z 方向直线移动，然后依次绕 X/Y/Z 轴旋转（全局坐标系） |
| **OSC_POSITION** | 手爪依次沿 X/Y/Z 方向直线移动（不旋转） |
| **IK_POSE** | 类似 OSC_POSE，但旋转是相对于手爪本地坐标系的 |
| **JOINT_POSITION** | 关节依次转动，手爪可能画弧线（不是直线） |
| **JOINT_VELOCITY** | 关节依次匀速转动 |
| **JOINT_TORQUE** | 关节施加扭矩，动作会比较"懒散"，因为 0 扭矩不能保证稳定 |

## 🔑 关键要点总结

1. **目的**：测试每个控制器的每个动作维度
2. **方法**：逐个维度施加测试值，观察机器人反应
3. **结构**：选择环境 → 选择机器人 → 选择控制器 → 执行测试循环
4. **输出**：可视化窗口显示机器人动作
5. **用途**：帮助开发者理解不同控制器的行为特性

这个脚本是学习 robosuite 控制器的绝佳起点！
