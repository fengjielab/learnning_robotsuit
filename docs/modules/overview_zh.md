# 模块概览

![模块概览图](../images/module_overview.png)

---

我们的框架提供两大类 API：

1. **建模 API（Modeling APIs）**：用于以模块化和可编程的方式定义仿真环境
2. **仿真 API（Simulation APIs）**：用于与外部输入进行交互，例如来自**策略（Policy）**或**输入/输出设备（I/O Device）**的输入

一个由建模 API 指定的**仿真模型（Simulation Model）**会被 **MuJoCo 引擎**实例化，创建一个称为**环境（Environment）**的仿真运行时。**环境**通过**传感器（Sensors）**（如相机和本体感知）生成观测值，并通过**机器人（Robots）**的**控制器（Controllers）**接收来自策略或设备的动作命令。上图说明了我们框架中的关键组件及其关系。

---

一个仿真模型由一个 **任务（Task）** 对象定义，该对象封装了机器人仿真的三个基本组成部分：

- **机器人模型（Robot Model）**
- **物体模型（Object Model）**
- **竞技场/场景（Arena）**

一个任务可以包含一个或多个机器人、零个或多个物体，以及单个场景。

**机器人模型**加载机器人的模型，还可以加载其他模型；例如，**机械臂（Manipulator）**机器人模型类还会从 XML 文件加载相应的**夹爪模型（Gripper Model）**。

**物体模型**可以从 3D 物体资产加载，也可以通过编程 API 程序化生成。

**场景（Arena）**定义了机器人的工作空间，包括环境固定装置（例如桌子）及其放置位置。

**任务类**将这些组成部分组合成一个 MuJoCo 的 **MJCF 建模语言**中的单一 XML 对象。这个 MJCF 对象通过 **mujoco** 库传递给 MuJoCo 引擎，以实例化 **MjModel** 对象用于仿真运行时。

---

**环境（Environment）** 对象提供 **OpenAI Gym** 风格的 API，供外部输入与仿真进行交互。

**外部输入**对应于用于控制**机器人**及其拥有的任何执行器的动作命令（例如，在机械臂机器人情况下，包括手臂关节和夹爪），其中运动学分量特定于机器人使用的**控制器**。

例如：
- 对于机器人机械臂的**关节空间控制器**，动作空间对应于机器人的关节数量
- 对于**操作空间控制器**，动作空间对应于 3D 笛卡尔运动或 6D 末端执行器位姿

这些动作命令可以：
- 由算法自动生成（例如深度神经网络策略）
- 或来自用于人类遥操作的 **I/O 设备**（例如键盘）

机器人的**控制器**负责解释这些动作命令，并将它们转换为传递给底层物理引擎的低层级扭矩，该引擎执行内部计算以确定仿真的下一步状态。

**传感器**从 MjSim 对象中提取信息并生成观测值，作为机器人对其动作的响应所接收的物理信号。

我们的框架支持多模态传感模式，例如：
- RGB-D 相机
- 力 - 扭矩测量
- 本体感知数据

并提供模块化 API 来建模真实的传感器动力学。

除了这些感官数据外，**环境**还提供有关任务进度和成功条件的附加信息，包括：
- 奖励函数（用于强化学习）
- 其他元数据
